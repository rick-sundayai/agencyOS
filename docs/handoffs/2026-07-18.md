## Key decisions (this session)
- Design system is **semantic CSS + design tokens, NO Tailwind** (AgencyOS has never had Tailwind ‚Äî an earlier mis-attribution to a RecruiterPro config was corrected). Both **light + dark** ship on a `data-theme` token layer; "color is the alarm." See ADRs.
- Reskins are **browser-verified, not unit-tested**; only genuinely new logic gets a seam (pure selectors for roster/health rail; extended RTL for card/queue/drawer/theme-toggle).
- Two **data gaps** surfaced (capabilities built + fixture-tested, but current data can't exercise them): the roster's **working/stalled** states need in-flight runs (but `insertAgentRun` always stamps `finished_at`), and the Drawer's **sourced/inferred** evidence needs provenance (but `ReasoningSchema.evidence` is bare `string[]`). Both need upstream data-model/instrumentation changes ‚Äî candidates for future tickets, relevant to what "production" must feed.
